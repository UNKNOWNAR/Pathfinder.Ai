Findings from Code Reuse Review:

1. **CSS Duplication in Views:**
   Almost all `.vue` files, including the newly added `CompanyDashboardView.vue` and `AdminCompaniesView.vue` duplicate the `.page` CSS class and its custom properties:
   ```css
   .page {
     --ink:     #111111;
     --bg:      #DEDEDE;
     --surface: #FFFFFF;
     --accent:  #2d8cf0;
     --border:  2px solid var(--ink);
     --shadow:  4px 4px 0 var(--ink);
     min-height: 100vh;
     background: var(--bg);
     color: var(--ink);
     font-family: 'Segoe UI', sans-serif;
   }
   ```
   **Actionable Finding:** Extract these common CSS variables and basic page layout styles into a global stylesheet (e.g., `src/assets/main.css` or `App.vue`'s non-scoped style block) to reduce duplication across views.

2. **UI Component Duplication (Buttons & Inputs):**
   The `.brutal-input`, `.brutal-textarea`, `.box`, `.status-msg`, and `.primary-btn` styling classes are redefined in multiple views (`CompanyDashboardView.vue`, `AdminHarvestersView.vue`, `HomeView.vue`, etc.).
   **Actionable Finding:** These are standard UI elements of the neo-brutalist theme and should be either extracted to a global CSS file or turned into reusable Vue components (e.g., `<BrutalInput>`, `<BrutalButton>`, `<BrutalBox>`).

3. **Status Message Logic:**
   In `CompanyDashboardView.vue`, there's inline logic for error/success message state:
   ```js
   const message = ref('');
   const isError = ref(false);
   ```
   **Actionable Finding:** Consider a global toast notification system or a reusable `<StatusMessage>` component to handle these API response states uniformly across forms.

4. **API Endpoints:**
   The newly introduced API endpoints (`/company/register`, `/company/jobs`, `/admin/companies`) use the centralized `api.js` Axios instance appropriately. No duplicated API logic was found.
